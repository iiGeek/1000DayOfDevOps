## üß† Operating System Concepts I Learned

### üîπ 1. What is a Process?
A process is a running program inside the OS.  
It receives:
- CPU time  
- Memory  
- Resources  

I learned how processes move through states:  
**New ‚Üí Ready ‚Üí Running ‚Üí Waiting ‚Üí Terminated**

### üîπ 2. Process vs Thread
- A **process** = a full program with its own memory  
- A **thread** = a lightweight worker inside the process  
- Processes are isolated, but threads share memory

### üîπ 3. CPU & Process Relationship
- CPU executes one thread at a time  
- OS switches between processes using ‚Äúcontext switching‚Äù  
- This explains why some programs lag or freeze

### üîπ 4. Why Programs Use RAM Differently
- Some apps load more data into memory  
- Background threads use extra memory  
- Heavy apps reserve large memory blocks  

### üîπ 5. CPU Scheduling Basics
I learned how the CPU decides *which process runs next* using algorithms:
- FCFS (First Come First Served)  
- SJF (Shortest Job First)  
- Round Robin  

These algorithms keep the system fast and responsive.
